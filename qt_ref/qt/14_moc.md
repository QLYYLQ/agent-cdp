# Using the Meta-Object Compiler (moc) - Qt 6.10

> Source: https://doc.qt.io/qt-6/moc.html

## Overview

The Meta-Object Compiler (moc) processes C++ header files containing the `Q_OBJECT` macro and generates corresponding meta-object code. This generated code enables signals/slots, runtime type information, and the dynamic property system.

## Core Functionality

The moc tool reads C++ header files and produces C++ source files when it detects class declarations with `Q_OBJECT`. "The C++ source file generated by moc must be compiled and linked with the implementation of the class."

Both qmake and CMake handle moc invocation automatically:
- **qmake**: Adds build rules by default
- **CMake**: Use the `AUTOMOC` property for automatic moc handling

## Usage Examples

### Basic Signal/Slot Class
```cpp
class MyClass : public QObject
{
    Q_OBJECT

public:
    MyClass(QObject *parent = 0);
    ~MyClass();

signals:
    void mySignal();

public slots:
    void mySlot();
};
```

### Properties with Q_PROPERTY and Q_ENUM
```cpp
class MyClass : public QObject
{
    Q_OBJECT
    Q_PROPERTY(Priority priority READ priority WRITE setPriority)

public:
    enum Priority { High, Low, VeryHigh, VeryLow };
    Q_ENUM(Priority)

    void setPriority(Priority priority) { m_priority = priority; }
    Priority priority() const { return m_priority; }

private:
    Priority m_priority;
};
```

## Command-Line Options

| Option | Purpose |
|--------|---------|
| `-D<macro>[=<def>]` | Define macro with optional definition |
| `-E` | Preprocess only; skip meta-object generation |
| `-f[<file>]` | Force `#include` statement in output |
| `-Fdir` | macOS framework directory search path |
| `-I<dir>` | Add directory to header include path |
| `-M<key=value>` | Append metadata to plugins |
| `-o<file>` | Write output to specified file |
| `-p<path>` | Prepend path to generated `#include` statement |

## Limitations and Restrictions

### Template Classes
Class templates cannot use `Q_OBJECT`:
```cpp
class SomeTemplate<int> : public QFrame
{
    Q_OBJECT  // INVALID
};
```

### Multiple Inheritance
"moc assumes that the first inherited class is a subclass of QObject. Also, be sure that only the first inherited class is a QObject."

```cpp
// Correct
class SomeClass : public QObject, public OtherClass { };

// Virtual inheritance with QObject is NOT supported
```

### Function Pointer Parameters
Function pointers cannot appear as signal/slot parameters:
```cpp
// WRONG
void apply(void (*apply)(List *, void *), char *);

// CORRECT - use typedef
typedef void (*ApplyFunction)(List *, void *);
void apply(ApplyFunction, char *);
```

### Fully Qualified Type Names
Enums and typedefs must be fully qualified in signal/slot declarations. "QObject::connect() compares the data types literally."

### Nested Classes
Nested classes cannot have signals or slots.

### Return Type Restrictions
"Signals and slots can have return types, but signals or slots returning references will be treated as returning void."

## Diagnostics

Common linkage errors:
- "YourClass::className() is undefined"
- "YourClass lacks a vtable"

Usually caused by forgetting to compile or `#include` moc-generated code.
